# jah

`jah` is a terminal UI for browsing markdown ticket files. It is built with
[Textual](https://textual.textualize.io/) and managed with `uv`.

The name is short for "jira-at-home": a lightweight way to browse local tickets
without turning a markdown workflow into a full task management system.

The initial target is ticket directories generated by
[hmans/beans](https://github.com/hmans/beans), while keeping the parser simple
enough for any flat directory of markdown files with YAML frontmatter.

## Features

- Tree view for parent/child ticket hierarchy
- Kanban view for leaf tickets
- Live search across title, tags, and body
- Hide-done filtering
- Detail preview pane plus a scrollable detail modal on `Enter`
- `$EDITOR` integration for external editing
- Clipboard copy for selected ticket IDs
- Keyboard help, status filtering, and kanban column toggles

## Install

Install dependencies with `uv`:

```bash
uv sync
```

## Usage

Run the TUI against a Beans directory or another flat markdown ticket directory:

```bash
uv run jah path/to/.beans
```

If no path is passed, `jah` reads tickets from the current directory.

## Keyboard

- `tab`: switch Tree / Kanban
- `/`: search
- `Esc`: clear search
- `?`: help
- `c`: choose visible kanban columns
- `s`: filter visible statuses
- `d`: toggle hide done
- `r`: refresh
- `y`: copy selected ticket ID
- `e`: edit selected ticket in `$EDITOR`
- `Enter`: open selected ticket detail modal

## Development

Run tests:

```bash
uv run pytest
```

Compile-check the Python files:

```bash
uv run python -m py_compile src/jah/*.py tests/*.py
```
